# OpenCode UI/UX 2026 — Flutter Integration Skill

This skill provides guidelines and best practices for integrating the OpenCode UI/UX 2026 design system within Flutter applications. It covers mapping design tokens to Flutter's `ThemeData`, implementing dark/light modes, handling navigation, displaying alerts, managing gestures, and adapting to platform-specific behaviors.

## 1. Design Tokens → ThemeData Mapping

Flutter's `ThemeData` is the central place to define the visual properties of your application. Mapping OpenCode design tokens (like `var(--accent)`, `var(--space-4)`) to `ThemeData` ensures consistency and allows for easy theming.

### Manual Mapping

For smaller projects or specific needs, you can manually map design tokens to `ThemeData` properties.

#### Step-by-step Instructions

1.  **Define a `ThemeData` object**: Create a `ThemeData` instance and populate its properties (colors, text styles, spacing) using values derived from your design tokens.
2.  **Use `Theme.of(context)`**: Access these properties throughout your widget tree using `Theme.of(context).colorScheme.primary` or `Theme.of(context).textTheme.bodyMedium`.

#### Code Example

```dart
// lib/theme/app_theme.dart
import 'package:flutter/material.dart';

class AppTheme {
  static final Color _primaryColor = Color(0xFF0066FF); // Corresponds to var(--accent)
  static final Color _backgroundColor = Color(0xFFFFFFFF); // Corresponds to var(--bg) light
  static final Color _textColor = Color(0xFF0F1117); // Corresponds to var(--text) light

  static ThemeData lightTheme = ThemeData(
    brightness: Brightness.light,
    primaryColor: _primaryColor,
    colorScheme: ColorScheme.light(
      primary: _primaryColor,
      background: _backgroundColor,
      onBackground: _textColor,
      surface: Color(0xFFF7F8FA), // Corresponds to var(--surface) light
    ),
    textTheme: TextTheme(
      bodyMedium: TextStyle(color: _textColor, fontSize: 16.0), // Corresponds to var(--text-base)
      // Map other text styles
    ),
    // Define spacing using constants or extensions
    // For example, using a custom extension for spacing:
    // extension BuildContextExtension on BuildContext {
    //   double get space1 => 4.0;
    //   double get space2 => 8.0;
    // }
  );

  static ThemeData darkTheme = ThemeData(
    brightness: Brightness.dark,
    primaryColor: Color(0xFF4D8EFF), // Corresponds to var(--accent) dark
    colorScheme: ColorScheme.dark(
      primary: Color(0xFF4D8EFF),
      background: Color(0xFF0A0B0E), // Corresponds to var(--bg) dark
      onBackground: Color(0xFFEDF0F5), // Corresponds to var(--text) dark
      surface: Color(0xFF13151A), // Corresponds to var(--surface) dark
    ),
    textTheme: TextTheme(
      bodyMedium: TextStyle(color: Color(0xFFEDF0F5), fontSize: 16.0),
    ),
  );
}

// Usage in a widget:
// Text(
//   'Hello OpenCode',
//   style: Theme.of(context).textTheme.bodyMedium,
// )
// Container(
//   color: Theme.of(context).colorScheme.primary,
//   padding: EdgeInsets.all(context.space4), // If using custom spacing extension
// )
```

### Using `token_theme_kit` (or similar packages)

For larger design systems, packages like `token_theme_kit` can automate the mapping of design tokens (often from JSON/YAML) to `ThemeData`.

#### Step-by-step Instructions

1.  **Add dependency**: Include `token_theme_kit` in your `pubspec.yaml`.
2.  **Define tokens**: Structure your design tokens in a JSON or YAML file, following the package's conventions.
3.  **Generate `ThemeData`**: Use the package's tools to generate `ThemeData` objects from your token definitions.

#### Code Example (Conceptual with `token_theme_kit`)

## 2. Dark/Light Mode via `ThemeProvider` and `useColorScheme`

Implementing dark and light modes is crucial for user preference and accessibility. Flutter provides robust mechanisms for this.

### Step-by-step Instructions

1.  **Wrap with `MaterialApp`**: Ensure your application is wrapped with `MaterialApp`.
2.  **Provide `theme` and `darkTheme`**: Assign your `lightTheme` and `darkTheme` (from `AppTheme` above) to the `theme` and `darkTheme` properties of `MaterialApp`.
3.  **Set `themeMode`**: Use `ThemeMode.system` to follow the system setting, or `ThemeMode.light`/`ThemeMode.dark` for explicit control.
4.  **Access current theme**: Use `MediaQuery.of(context).platformBrightness` or `Theme.of(context).brightness` to determine the current theme.

#### Code Example

```dart
// lib/main.dart
import 'package:flutter/material.dart';
import 'package:your_app/theme/app_theme.dart';

void main() {
  runApp(const MyApp());
}

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: 'OpenCode UI/UX',
      theme: AppTheme.lightTheme, // Default light theme
      darkTheme: AppTheme.darkTheme, // Dark theme
      themeMode: ThemeMode.system, // Follow system theme preference
      home: const MyHomePage(),
    );
  }
}

class MyHomePage extends StatefulWidget {
  const MyHomePage({super.key});

  @override
  State<MyHomePage> createState() => _MyHomePageState();
}

class _MyHomePageState extends State<MyHomePage> {
  @override
  Widget build(BuildContext context) {
    final isDarkMode = Theme.of(context).brightness == Brightness.dark;
    return Scaffold(
      appBar: AppBar(
        title: Text(
          'OpenCode App',
          style: TextStyle(color: Theme.of(context).colorScheme.onPrimary),
        ),
        backgroundColor: Theme.of(context).colorScheme.primary,
      ),
      body: Center(
        child: Text(
          isDarkMode ? 'Hello Dark Mode!' : 'Hello Light Mode!',
          style: Theme.of(context).textTheme.bodyMedium,
        ),
      ),
    );
  }
}
```

## 3. Navigation (go_router or Navigator 2.0 Patterns)

Flutter offers powerful navigation solutions. `go_router` is a popular package that simplifies complex navigation flows, while Navigator 2.0 provides fine-grained control.

### Using `go_router`

`go_router` is recommended for most applications due to its declarative nature and deep linking support.

#### Step-by-step Instructions

1.  **Add dependency**: Include `go_router` in your `pubspec.yaml`.
2.  **Define routes**: Create a `GoRouter` instance with your application's routes.
3.  **Integrate with `MaterialApp.router`**: Use `MaterialApp.router` and provide the `routerConfig`.

#### Code Example

```dart
// lib/main.dart (continued)
import 'package:go_router/go_router.dart';

// ... (MyApp and AppTheme definitions)

final _router = GoRouter(
  routes: [
    GoRoute(
      path: '/',
      builder: (context, state) => const HomeScreen(),
    ),
    GoRoute(
      path: '/details/:id',
      builder: (context, state) => DetailScreen(id: state.pathParameters['id']!),
    ),
  ],
);

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp.router(
      title: 'OpenCode UI/UX',
      theme: AppTheme.lightTheme,
      darkTheme: AppTheme.darkTheme,
      themeMode: ThemeMode.system,
      routerConfig: _router,
    );
  }
}

class HomeScreen extends StatelessWidget {
  const HomeScreen({super.key});

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('Home')),
      body: Center(
        child: ElevatedButton(
          onPressed: () => context.go('/details/123'),
          style: ElevatedButton.styleFrom(
            backgroundColor: Theme.of(context).colorScheme.primary, // Using design token
            padding: EdgeInsets.symmetric(horizontal: 24, vertical: 12), // Manual spacing for example
          ),
          child: Text(
            'Go to Details',
            style: TextStyle(color: Theme.of(context).colorScheme.onPrimary),
          ),
        ),
      ),
    );
  }
}

class DetailScreen extends StatelessWidget {
  final String id;
  const DetailScreen({super.key, required this.id});

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: Text('Details for $id')),
      body: Center(child: Text('Detail Screen for ID: $id')),
    );
  }
}
```

## 4. Alerts and Toasts

Flutter provides built-in widgets for alerts and toasts (`SnackBar`). For more complex or custom alerts, you can use `showDialog`.

### Using `SnackBar` for Toasts

`SnackBar` is ideal for displaying brief messages to the user at the bottom of the screen.

#### Step-by-step Instructions

1.  **Access `ScaffoldMessenger`**: Use `ScaffoldMessenger.of(context)` to show a `SnackBar`.
2.  **Create `SnackBar`**: Instantiate `SnackBar` with content and optional actions.

#### Code Example

```dart
// In a widget's build method or event handler
void _showToast(BuildContext context) {
  ScaffoldMessenger.of(context).showSnackBar(
    SnackBar(
      content: Text(
        'Action completed successfully!',
        style: TextStyle(color: Theme.of(context).colorScheme.onPrimary), // Using design token
      ),
      backgroundColor: Theme.of(context).colorScheme.primary, // Using design token
      duration: const Duration(seconds: 3),
      action: SnackBarAction(
        label: 'UNDO',
        onPressed: () {
          // Some action to undo
        },
        textColor: Theme.of(context).colorScheme.onPrimary,
      ),
    ),
  );
}

// Call it:
// ElevatedButton(
//   onPressed: () => _showToast(context),
//   child: const Text('Show Toast'),
// )
```

### Using `showDialog` for Alerts

`showDialog` is used for modal alerts that require user interaction.

#### Step-by-step Instructions

1.  **Call `showDialog`**: Pass a `BuildContext` and a `builder` function that returns an `AlertDialog`.
2.  **Customize `AlertDialog`**: Define title, content, and actions using `TextButton` or `ElevatedButton`.

#### Code Example

```dart
// In a widget's build method or event handler
void _showAlert(BuildContext context) {
  showDialog(
    context: context,
    builder: (BuildContext context) {
      return AlertDialog(
        title: Text(
          'Confirm Action',
          style: Theme.of(context).textTheme.titleLarge,
        ),
        content: Text(
          'Are you sure you want to proceed?',
          style: Theme.of(context).textTheme.bodyMedium,
        ),
        actions: <Widget>[
          TextButton(
            onPressed: () => Navigator.of(context).pop(),
            child: Text(
              'Cancel',
              style: TextStyle(color: Theme.of(context).colorScheme.onSurface),
            ),
          ),
          ElevatedButton(
            onPressed: () {
              Navigator.of(context).pop();
              // Perform action
            },
            style: ElevatedButton.styleFrom(
              backgroundColor: Theme.of(context).colorScheme.error, // Example: using an error token
            ),
            child: Text(
              'Delete',
              style: TextStyle(color: Theme.of(context).colorScheme.onError),
            ),
          ),
        ],
      );
    },
  );
}

// Call it:
// ElevatedButton(
//   onPressed: () => _showAlert(context),
//   child: const Text('Show Alert'),
// )
```

## 5. Gestures (Swipe to Dismiss, Pull to Refresh)

Flutter provides rich gesture detection capabilities, making it easy to implement common UI interactions.

### Swipe to Dismiss

Used for dismissing list items or other dismissible widgets.

#### Step-by-step Instructions

1.  **Wrap with `Dismissible`**: Wrap the widget you want to make dismissible with a `Dismissible` widget.
2.  **Provide `key` and `onDismissed`**: A unique `key` is required, and `onDismissed` callback handles the logic after dismissal.

#### Code Example

```dart
// In a ListView.builder or similar
ListView.builder(
  itemCount: _items.length,
  itemBuilder: (context, index) {
    final item = _items[index];
    return Dismissible(
      key: Key(item), // Unique key for each item
      onDismissed: (direction) {
        setState(() {
          _items.removeAt(index);
        });
        ScaffoldMessenger.of(context).showSnackBar(
          SnackBar(content: Text('$item dismissed')),
        );
      },
      background: Container(color: Colors.red), // Background when swiping
      child: ListTile(title: Text(item)),
    );
  },
)
```

### Pull to Refresh

Commonly used in lists to fetch new data.

#### Step-by-step Instructions

1.  **Wrap with `RefreshIndicator`**: Wrap your scrollable widget (e.g., `ListView`, `CustomScrollView`) with `RefreshIndicator`.
2.  **Provide `onRefresh`**: Implement the asynchronous `onRefresh` callback to fetch new data.

#### Code Example

```dart
// In a StatefulWidget
Future<void> _handleRefresh() async {
  // Simulate network request
  await Future.delayed(const Duration(seconds: 2));
  setState(() {
    _items.add('New Item ${DateTime.now().second}');
  });
}

// In build method
RefreshIndicator(
  onRefresh: _handleRefresh,
  child: ListView.builder(
    itemCount: _items.length,
    itemBuilder: (context, index) {
      return ListTile(title: Text(_items[index]));
    },
  ),
)
```

## 6. Platform-Specific Adaptations (Android vs iOS)

Flutter allows you to build a single codebase for multiple platforms, but sometimes platform-specific UI/UX adjustments are necessary.

### Step-by-step Instructions

1.  **Use `Theme.of(context).platform`**: Check `TargetPlatform` to determine the current platform.
2.  **Conditional Widgets**: Render different widgets or apply different styles based on the platform.
3.  **Platform-specific packages**: Utilize packages like `cupertino_icons` for iOS-style icons or `flutter_platform_widgets` for adaptive widgets.

#### Code Example

```dart
import 'package:flutter/material.dart';
import 'package:flutter/cupertino.dart'; // For iOS-style widgets

class PlatformAdaptiveAppBar extends StatelessWidget implements PreferredSizeWidget {
  final String title;

  const PlatformAdaptiveAppBar({super.key, required this.title});

  @override
  Widget build(BuildContext context) {
    final TargetPlatform platform = Theme.of(context).platform;

    if (platform == TargetPlatform.iOS) {
      return CupertinoNavigationBar(
        middle: Text(title),
        backgroundColor: CupertinoColors.systemBackground.resolveFrom(context),
        // Use design tokens for colors if mapped to CupertinoThemeData
      );
    } else {
      return AppBar(
        title: Text(title),
        backgroundColor: Theme.of(context).colorScheme.primary, // Using design token
      );
    }
  }

  @override
  Size get preferredSize => const Size.fromHeight(kToolbarHeight);
}

// Usage:
// Scaffold(
//   appBar: PlatformAdaptiveAppBar(title: 'My App'),
//   body: ...
// )
```
